# Исследовательский суперкомпилятор для модельного языка

## 📌 Описание
Этот проект — учебный прототип **суперкомпилятора** для модельного языка.  
Язык минимален: функции и конструкторы, сопоставление с образцом только по первому аргументу, **без встроенной арифметики** (но с поддержкой числовых литералов).
Результаты суперкомпиляции:
- граф конфигураций (с экспортом в DOT/JSON),
- остаточная программа (pretty-print на исходном языке).

Проект выполняется как часть курсовой работы и служит для сравнения стратегий суперкомпиляции.

---

## 🎯 Цели проекта
1. **Реализация суперкомпилятора** для модельного языка:
   - парсер, pretty-printer, проверки корректности;
   - драйвинг (symbolic driving), построение графа;
   - контроль зацикливания (обобщение, обратные дуги);
   - резидуализация (остаточная программа).
2. **Сравнение подходов**:
   - перестройка **сверху** vs перестройка **снизу** (гиперцикл С.М. Абрамова);
   - проверка на зацикливание: **Хигмана–Крускала** vs **мешок тегов**.
3. **Эксперименты**:
   - найти примеры программ, где стратегии ведут себя по-разному;
   - интерпретировать различия (размер графа, точность специализации и т.д.).

## 🛠️ Архитектура и структура
Проект организован как Python-пакет `sll`. Основные модули:

### 📂 sll/ (Ядро)
**Frontend:**
- **`ast_nodes.py`**: Определения узлов AST. Узлы поддерживают tag для работы стратегии Bag of Tags.
- **`parser.py`**: Рекурсивный спуск для разбора грамматики SLL.
- **`type_checker.py`**: Семантический анализатор. Проверяет корректность типов, конструкторов и арности перед запуском.
- **`preprocessor.py`**: Тэггер. Проставляет уникальные метки (теги) на узлы программы перед запуском (необходимо для стратегии Bag of Tags).

**Core (Суперкомпиляция):**
- **`driver.py`**: "Двигатель" прогонки. Реализует приоритетную логику правил и поддержку вложенных вызовов.
- **`supercompiler.py`**: Главный цикл суперкомпиляции. Управляет построением дерева, вызывает драйвер, проверяет свистки и выполняет свертку (Folding).
- **`process_tree.py`**: Структура дерева процессов (узлы, контексты типов, обратные ссылки).
- **`he.py`**: Свисток #1. Реализация алгоритма **Гомеоморфного вложения** (HE).
- **`bag_of_tags.py`**: Свисток #2. Реализация алгоритма **Мешок тегов** (Bag of Tags).
- **`msg.py`**: Алгоритм наиболее конкретного обобщения с поддержкой CSE (Common Subexpression Elimination).

**Backend:**
- **`residualizer.py`**: Преобразователь графа в новую SLL-программу с выделением f- и g-функций.
- **`exporter.py`**: Визуализация. Экспортирует граф процесса в формат DOT/Graphviz.


## 🚀 Запуск и использование

Для работы требуется **Python 3.10+** (используется синтаксис `match/case`).

### Установка зависимостей
Для генерации картинок графов установите библиотеку `graphviz`:
```bash
pip install graphviz
```


### Запуск суперкомпилятора
Пример команды для запуска суперкомпилятора:
```bash
python3 main.py <путь_к_файлу> <имя_функции_или_выражение> [аргументы]
```
Параметры:

- -t / --types: Типы свободных переменных (например, a=Nat).
- -s / --strategy: Стратегия контроля зацикливания:
  - HE — Гомеоморфное вложение (по умолчанию).
  - TAG — Мешок тегов (Bag of Tags).
- -o / --out: Имя выходного файла (без расширения) для сохранения графа и картинки.
- -g / --gen: Выбор перестройки - TOP или BOTTOM (по умолчанию TOP).
- -d / --dev: Включить режим разработчика (отображение тэгов) - ON/OFF (по умолчанию OFF).

### Пример
```bash
python3 main.py commute.sll main -o commute/he
```
